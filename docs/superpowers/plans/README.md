# superpowers plans 目录说明

本目录存放执行计划文档，不作为“当前已生效规范”的唯一来源。

## 状态标记规则

每个计划文件标题下应增加一行状态标记，格式如下：

`> 状态：草案 | 执行中 | 已完成 | 已归档`

建议同时附加更新时间与适用范围，例如：

`> 状态：草案（仅方案，未完全落地）`

## 与其他目录的边界

- `docs/design/99-tools/`：长期设计、运行规则、不变量（偏稳定）。
- `docs/superpowers/plans/`：阶段计划与实施路线（会迭代）。
- `docs/authoritative-requirements/`：需求基线（权威来源）。

## 使用建议

- 引用计划文档时，请同时标注状态，避免将草案当作已落地实现。
- 计划执行后，应补充“已完成/已归档”状态并记录完成日期。
